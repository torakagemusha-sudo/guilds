Design Primitives

The Invariant Foundation of GUI Layout — Atomic, Irreducible, Universal

⚡ The Foundation

All graphical user interfaces, from the first Alto to spatial computing, are composed of invariant primitives — fundamental concepts that exist independent of technology, platform, or era. These are not trends or patterns, but the irreducible elements of human-computer interaction.

The Conservation of Complexity
Complexity cannot be destroyed, only displaced. A simplified interface pushes complexity to the backend, the user's mind, documentation, or training. Design is the conscious allocation of this complexity.

⏱ The Latency Thresholds
0-100ms : Instant, feels direct (perceptual fusion)
100-1000ms: Noticeable delay, still fluid (keep under 1s)
1-10s : Mental context switch risk (provide progress)
>10s : Abandonment threshold (require async handling)
>10m : Task failure, session end (state preservation critical)

🧠 The Perceptual Hierarchy
Users perceive in this order, regardless of designer intent:
1. Motion (change over time) — fastest detection
2. Color (hue, saturation, value) — emotional impact
3. Contrast (figure/ground) — boundary detection
4. Size (relative scale) — importance signaling
5. Position (spatial relationship) — structure parsing
6. Typography (symbolic content) — slowest, requires literacy

⚖️ Universal Laws

The immutable constraints governing all interface design. Violate these at your peril — they are backed by physics, physiology, and psychology.

1. Law of Composition
Primitives compose but do not blend.
Container + Hierarchy = Nested containment
Transition + Feedback = Confident interaction
Metaphor + Convention = Cultural interface
Complex behaviors emerge from primitive combinations, but each primitive maintains its distinct properties. A container within a hierarchy is still a container first.
2. Law of Conservation
Complexity cannot be destroyed, only displaced.

L_interface + L_backend + L_user + L_docs = Constant

Where L = cognitive load
A simplified interface pushes complexity elsewhere: to the backend (computation), the user's mind (mental models), documentation (external memory), or training (prior knowledge).
3. Law of Mode Danger
N binary modes → 2^N possible states
User must track: current mode + available transitions

Invisible modes → mode errors (the "oops" factor)
Modes multiply state space exponentially. The user must maintain awareness of current mode and available transitions. Invisible modes cause mode errors — performing the right action in the wrong mode.
4. Law of Feedback
Action + Feedback = Control
Action − Feedback = Anxiety

Feedback must be: Immediate (<100ms), Informative, Consistent
Every action requires an equal and opposite reaction. Without feedback, users cannot confirm their action registered, leading to repeated inputs or abandonment.
5. Law of Hierarchy Parsing
Users unconsciously construct mental models from structural primitives. Ambiguous hierarchy creates parse errors — the user cannot determine relationships and thus cannot predict behavior.
6. Design as Constraint Satisfaction
Given:
P = Set of primitives
C = Constraints (user, business, technical)
G = Goals (usability, delight, efficiency)

Find: Layout L ⊆ P such that ∀c ∈ C: satisfies(L, c) ∧ optimizes(L, G)

Subject to:
Cognitive Load ≤ Working Memory Capacity
Feedback Latency ≤ Attention Threshold
Mode Count ≤ Mental Model Complexity Limit

Composition Rules
Valid Composition: Primitives combine if they share a dimension (Spatial + Structural = Layout) or form a causal chain (Trigger → Transition → State).

Invalid Composition: Primitives conflict if they violate conservation (Infinite Container + Finite Space) or create ambiguity (Parallel + Sequence for same elements).

✓ The Irreducible Checklist

Every GUI layout, regardless of complexity, must resolve these six dimensions. Failure at any primitive level propagates upward — a layout cannot succeed if its temporal transitions violate attention limits, regardless of spatial elegance.

  • Space — Where does each element exist?
    Containers defined, regions bounded, layers ordered
  • Structure — How do elements relate?
    Hierarchy parseable, sequence logical, proximity meaningful
  • Time — How do elements change?
    States defined, transitions smooth, feedback within 100ms
  • Meaning — What do elements signify?
    Signifiers visible, affordances clear, mappings natural
  • Action — How do elements behave?
    Activation clear, selection unambiguous, navigation orienting
  • Mind — How do users process?
    Load minimized, chunking supported, recognition > recall
Validation Criteria
A layout is valid when all six dimensions are addressed.
A layout is successful when all six dimensions are optimized.
A layout is innovative when it discovers new primitive combinations without violating invariant laws.
